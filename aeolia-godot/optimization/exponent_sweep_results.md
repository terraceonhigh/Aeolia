## Critical Diagnostics

### DF suppression across all modified approaches

Every non-baseline configuration produces `df_year=None` (Dark Forest never fires on seed 216089).
This is a calibration-breaking failure: `dark_forest_timing` returns a raw 4.0 penalty
(weight 2.0 → **8 loss points**) when DF doesn't fire. The baseline fires DF at -700 BP,
which is outside the target window [-200, -40] and incurs ~10 loss points via the quadratic timing
penalty — so the non-baseline approaches are marginally better on this term, but for the wrong reason.

**Root cause:** any non-zero `L_exp` applied to `core_pop` snowballs as polities expand.
A polity controlling 8 archipelagos with pop ~50 each has `core_pop ≈ 400`, giving
`(400/100)^0.5 = 2×` tech-growth multiplier at L=0.5. This drives the leading polity to absorb
all competitors before the awareness system builds to the 0.2 threshold needed for DF, leaving
one global hegemon with no peer to break against.

### THREE-REGIME naphtha scramble at -16,400 BP

Scramble fires at tick 72 (year -16,400) — the Mesolithic. The `pop^0.5` multiplier in the
Medieval and Industrial bands causes explosive growth that defeats the purpose of the Malthusian
floor: the floor slows tech 0-3, but the medieval/industrial pop exponent then rockets past it.

### Hegemon tech hitting 11.0

All modified approaches pin `hegemon_tech = 11.0` (soft ceiling). Baseline reaches 9.9 — near-nuclear
but not saturated. Hitting the ceiling compresses late-game narrative (DF, nuclear fleets, fleet
asymmetry).

## Verdict

**The Malthusian clamp at low tech is not the key differentiator. The uncalibrated population
exponent at higher tech eras is the dominant — and destructive — effect.**

THREE-REGIME nominally achieves the lowest mean loss (15.89 vs 25.25 baseline), but does so by:
- Accidentally scoring better on geography/hegemon terms on specific seeds
- Failing catastrophically on DF timing and naphtha scramble timing
- Hitting the tech ceiling, saturating all late-game dynamics

The gap between INTERPOLATED and THREE-REGIME is **7.4 loss units (29% of baseline)**. Ordinarily
this would suggest discrete regimes are worth the complexity, but here the gap is driven by the
population exponent, not the Malthusian boundary itself.

**Can we skip discrete regime boundaries?**
Yes, but not by adding a population exponent to the accelerator. The existing `accel_rate` discrete
table (0 / 0.002 / 0.008 / 0.025 / 0.120) already encodes five implicit regimes. INTERPOLATED
between K/E (no L term) achieves modest improvement on some seeds without breaking DF.

---

## Recommendation

**Do not add `pop^β` to the tech-growth accelerator.** Population already enters production
indirectly via `energy_surplus = crop_y × tp × 0.01` → budget → expansion. Adding it explicitly
double-counts and breaks competitive balance.

**Most promising hybrid** (not tested in this sweep, worth trying):
- Keep `L_exp = 0` throughout (no population term in accelerator)
- Apply Malthusian floor to **energy_surplus only** (Stage 1) for tech < 4:
  `surplus = min(carry_cap, tp) / tp × crop_y × tp × 0.01`
  This caps surplus when population is near carrying capacity, slowing expansion pacing
  without touching tech growth speed — so DF timing is preserved.
- Vary only K_exp and E_exp per era: e.g. `(K=0.20, E=0.80)` at low tech (land-constrained),
  `(K=0.35, E=0.65)` at industrial (energy-constrained). The coarse FIXED grid shows K≈0.30,
  E≈0.70 is close to optimal on seed 216089 (14.97 loss), consistent with the baseline's K=0.30.
