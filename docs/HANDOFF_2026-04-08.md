## 2. What Aeolia Is

A civilization simulation engine modeling a 4.6x Earth circumference ocean world (~95% water, no continuous landmass). Archipelago-structured. The simulation runs a 10,000-year tick-based history producing trade networks, resource scrambles, and nuclear deterrence equilibria.

**Framed as a game / thought experiment, NOT a research tool.** This is a product positioning decision from Terrace's faculty advisor. Even underspecified models get read as prescriptive — frame it as a game.

**Repository:** `~/Labs/Aeolia` on both Neo and Aomori. GitHub: `terraceonhigh/Aeolia` (public).

Read `Aeolia/CLAUDE.md` first. It has the full repo layout, sim architecture, running instructions, and conventions.

## 3. Architecture at a Glance

### Simulation Engine (`aeolia-godot/optimization/sim_proxy_v2.py`)

**SimParams:** 26 tunable parameters in a dataclass. All optimizer work targets this space.

**Tick pipeline (50-year resolution):**
1. Energy budget: crop yield x land + fish yield x coast = total calories. Malthusian clamp for tech < 4.
2. Culture & allocation: 2D continuous culture space (Collective<->Individual x Inward<->Outward). Drift based on prosperity, crisis, trade exposure, resource stress, fishery type.
3. Expansion: controller map, conquest, absorption blending (0.95 core + 0.05 target).
4. Detection & awareness: Dark Forest detection between nuclear-capable polities.
5. Tech growth & decay: accel_rate table (5 regimes). Maintenance cost = tech^2 x rate. Tech decays when surplus < maintenance.
6. Desperation: resource pressure overrides allocation. Food/industrial/nuclear deficit hierarchy.

**Key mechanics:**
- Three-threshold resources: Detection (geology) -> Exploitation (tech-gated) -> Strategic Valuation (event-triggered)
- Naphtha scramble (tech ~5), pyra scramble (tech ~8), Dark Forest (two nuclear hegemons detect each other)
- Desperation cascade: surplus shortfall -> tech decay -> allocation override -> aggressive expansion

### Worldbuilding

11 series bible documents in `docs/` (00-10). Design spec at `docs/DESIGN_SPEC.md` (1,300+ lines, the technical source of truth). Commodity names are canon from V1 sim_proxy.py maps — do not anglicize them.

### Godot Frontend

`aeolia-godot/` has a Godot 4 project structure. Migration plan in `dev/HANDOFF.md`. Agent instructions in `dev/AGENTS.md`. The monolith reference (React/Three.js, 2,282 lines) is at `src/MONOLITH_REFERENCE.jsx`.

## 8. Conventions and Don'ts

- Commodity names are canon from V1 sim_proxy.py maps (paddi, emmer, taro, sago, papa, nori, qahwa, char, kapas, seric, etc.)
- Fish species: sthaq (salmon), saak (eulachon), tunnu (tuna), sardai (sardine), bakala (cod), kauri (shellfish)
- Names drawn from historical non-English trade languages — do NOT anglicize
- Same seed must always produce same world. All randomness flows from mulberry32(seed). Never use Math.random() or built-in RNG.
- Simulation produces data. Rendering consumes it. They never mix.
- Status is derived, not assigned. Political statuses are labels for regions in continuous sovereignty-trade space.
- In mode-of-production space: labor_commodification <= surplus_centralization + 0.3 (hard constraint)
- Don't implement HOOK or OPEN QUESTION items from the design spec without discussing with Terrace first
- Don't run persistent agents on Neo — that's her study machine
